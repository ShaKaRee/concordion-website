Specs that make sense

By using lightweight, automated examples instead of long specification documents, the code you'll write will be fit for purpose, first time. Any changes to the spec will be version controlled - and will break the build if they no longer match with what the system actually does.

Fast feedback on every check-in

By pairing Concordion & SBE with a build orchestrator, you can run the tests whenever a change is made to version control.

See what's working

Concordion's living documentation is generated whenever the tests are run, and posted to a webpage of your choosing - so it's easy to be up to date. Tests are highlighted inline depending on their execution status. See the progress as developers write code to make tests pass and turn the specification green.

Up to date, revision controlled

Tests and the specification they're linked to are kept in version control, right next to the source code. It never gets out of date, and is easier to maintain in the long term.
